2.1 simulate_population function

The heart of the squid R package is the simulate_population function, which we can use to simulate hierarchical population level data. We provide the function with a set of parameters, a hierarchical data structure (if we are simulating hierarchical data), and various other optional arguments, which are listed below.

The simulate_population function simulates predictors at each hierarchical level, using provided mean and vcov parameters, from a multivariate normal distribution. These predictors are then scaled by the beta parameters, and added together to create the response. The arguments that can be provided to the simulate_population function (along with their defaults) are:

simulate_population(
  data_structure, 
  N, 
  parameters, 
  N_response=1, 
  family="gaussian", 
  link="identity", 
  model, 
  known_predictors, 
  pedigree, 
  pedigree_type, 
  phylogeny, 
  phylogeny_type, 
  cov_str, 
  N_pop=1
)

Each of these will be covered in more detail in the following sections. Briefly, N and data_structure refer to the size and structure of the data being simulated - data_structure is covered in more detail in Section 3. parameters is a list of parameters to be used in the simulation and is described in detail in the following parts of this section. N_response refers the number of response variable to be simulated and is covered in detail in the section on multivariate models (4). family and link refer to simulating non Gaussian response variables and are covered in Section 2.7. model allows for the specification of more complex models and is covered in Section 2.8.

pedigree and pedigree_type relate to simulating genetic effects and are covered in Section 5, phylogeny and phylogeny_type, relate to simulating phylogenetic effects and are covered in Section 6and cov_str relates to simulating a general covariance structure and is covered in multiple sections, including 5, 6, 7.3 and 7.4.

N_pop relates to the number of populations, or datasets, that you want to simulate for each parameter set. This is covered in Section 6
